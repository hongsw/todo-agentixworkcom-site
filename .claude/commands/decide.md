# /decide — 결정 요청 상세

특정 항목에 대해 의사결정자가 판단할 수 있도록 **상황 + 선택지**를 제시한다.

## 사용법
```
/decide {번호 또는 Task명}
```

## 실행할 것

### 1. 해당 TODO 항목의 전체 정보를 Notion에서 가져온다
- Task명, Status, Priority, Phase, 작업 요약, 코멘트

### 2. 고객 코멘트가 있으면 함께 가져온다

### 3. 아래 형식으로 출력

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 결정 필요: {Task명}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

상황:
{지금 이 항목이 왜 당신의 결정을 기다리고 있는지 2-3문장으로 설명}

{고객 코멘트가 있으면}
💬 고객 피드백:
"{원문 그대로}"

선택지:
  A) {옵션 설명} → 예상 결과/소요
  B) {옵션 설명} → 예상 결과/소요
  C) {옵션 설명} → 예상 결과/소요 (필요시)

추천: {근거와 함께 1문장}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
→ /approve {번호}
→ /reject {번호} "사유"
→ 또는 직접 지시: "B로 진행해"
```

### 4. 규칙
- "어떻게 할까요?"라고 묻지 마라. 반드시 선택지를 만들어라.
- 선택지마다 예상되는 결과와 소요(시간/비용)를 명시하라.
- 추천이 있으면 근거와 함께 제시하되, 최종 결정은 의사결정자에게 맡긴다.
- 기술 디테일은 빼라. "React 상태 관리를 변경해야"가 아니라 "화면 전환 방식을 바꿔야"로 써라.
