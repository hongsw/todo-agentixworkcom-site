# /dashboard — 현황판

의사결정자를 위한 현황 요약을 보여준다. 실행 디테일은 빼고 **판단에 필요한 것만** 보여준다.

## 지금 실행할 것

### 1. Notion TODO DB 전체 조회
모든 TODO 항목을 가져와서 Status별로 분류한다.

### 2. 아래 형식으로 출력

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 프로젝트 현황 — {오늘 날짜}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ 진행률: ██████████░░░░ {완료}/{전체} ({퍼센트}%)

✅ 완료  {N}건
🔄 진행중 {N}건
👀 리뷰중 {N}건 ← 고객 확인 대기
📋 대기  {N}건
⏸️ 보류  {N}건

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔔 당신이 확인할 것 ({M}건)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

{있으면 아래 항목들을 나열}

1. 👀 [{Task명}] — 작업 완료됨. 결과 확인 필요.
   요약: {작업 요약}
   → /approve 1 또는 /reject 1 "사유"

2. ⏸️ [{Task명}] — 기술 문제로 보류됨. 방향 결정 필요.
   상황: {왜 보류되었는지}
   → /decide 2

3. 💬 [{Task명}] — 고객이 코멘트를 남김. 대응 필요.
   고객: "{코멘트 요약}"
   → /decide 3

{없으면}
✨ 확인할 사항 없음. 모든 것이 정상 진행 중입니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 다음 실행 예정 (상위 3건)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. [{Priority}] {Task명} — {Phase}
2. [{Priority}] {Task명} — {Phase}
3. [{Priority}] {Task명} — {Phase}

💡 우선순위 변경: /reprioritize
🚀 Loop 실행: /kickoff
```

### 3. 규칙
- 기술 용어 쓰지 마라. 의사결정자는 코드를 모른다.
- 길게 쓰지 마라. 한 항목 최대 2줄.
- "결정 필요" 항목이 있으면 반드시 선택지를 제시하라.
- 결정 필요 없으면 "확인할 사항 없음"으로 깔끔하게 끝내라.
