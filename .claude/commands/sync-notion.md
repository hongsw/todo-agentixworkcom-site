# /sync-notion — 현재 작업을 Notion에 동기화

현재 Git 브랜치의 작업 내용을 Notion TODO DB에 동기화합니다.
수동으로 코드 작업을 한 후, Notion 기록을 업데이트할 때 사용합니다.

## 실행 순서

1. **현재 Git 상태 확인**: 현재 브랜치명, 최근 커밋 메시지, 변경된 파일 목록을 파악합니다.

2. **관련 TODO 찾기**: 브랜치명이나 커밋 메시지에서 TODO와 관련된 키워드를 추출하여 Notion TODO DB에서 매칭되는 항목을 찾습니다.

3. **PR 확인**: 현재 브랜치에 대한 열린 PR이 있는지 확인합니다.

4. **Notion 업데이트**:
   - 매칭되는 TODO가 있으면:
     - Status → "👀 리뷰중" (PR이 있는 경우) 또는 "🔄 진행중" (아직 작업 중)
     - PR 링크 → PR URL (있으면)
     - 작업 요약 → 변경사항을 **비개발자 언어**로 요약하여 기록
     - 완료일 → 오늘 (PR이 있는 경우)
   - 매칭되는 TODO가 없으면:
     - 새 TODO를 생성할지 사용자에게 확인
     - 생성 시 자동으로 Phase, Priority 추천

5. **결과 출력**: 업데이트된 Notion TODO 링크와 변경 내용을 표시합니다.

## 사용 예시

```
> /sync-notion
현재 브랜치: feature/dev-hero-section
관련 TODO: "히어로 섹션 UI/UX 디자인 및 퍼블리싱"
→ Notion Status: 🔄 진행중 → 👀 리뷰중
→ 작업 요약 업데이트 완료
→ PR 링크 추가 완료
```

## 주의사항

- CLAUDE.md의 작업 요약 작성 가이드를 반드시 따릅니다
- 기술 용어는 PR에만 사용하고, Notion에는 쉬운 말로 씁니다
