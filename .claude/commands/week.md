# /week — 주간 요약

의사결정자용 주간 요약 + Notion 프로젝트 페이지에 고객용 주간 리포트 자동 게시.

## 실행할 것

### 1. Notion TODO DB에서 이번 주 데이터 수집
- 이번 주 완료된 항목 (완료일 기준)
- 현재 진행중/리뷰중 항목
- 보류 항목과 사유
- 다음 주 예정 (대기 중 상위 항목)

### 2. 의사결정자에게 출력 (터미널)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 주간 요약 — {MM/DD ~ MM/DD}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

이번 주: {완료}건 완료 / {진행}건 진행중 / {보류}건 보류

주요 성과:
• {가장 중요한 완료 항목 1-3개, 한 줄씩}

주의 사항:
• {보류/이슈 항목. 없으면 "없음"}

다음 주 계획:
• {대기 상위 3건}

📈 전체 진행률: {퍼센트}% ({완료}/{전체})
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Notion에 고객용 리포트를 게시할까요? (Y/n)
```

### 3. 승인하면 Notion 프로젝트 페이지에 고객용 리포트 추가

고객용 리포트는 기술 용어 없이, 더 친절한 톤으로 작성:

```markdown
## 📊 주간 리포트 — {날짜}

### 이번 주 완료
- {Task명}: {작업 요약 1줄}

### 진행 중
- {Task명}: {현재 상태}

### 다음 주 예정
- {Task명}

### 확인 요청 드리는 사항
- {고객 확인 필요 항목}
```
