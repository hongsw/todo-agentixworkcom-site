# /reject — 반려

의사결정자가 항목을 반려한다. 너(Claude Code)가 자동으로 수정 TODO를 생성한다.

## 사용법
```
/reject {번호} "사유"
/reject {번호}              ← 사유를 물어본다
```

## 실행할 것

1. 사유가 없으면 의사결정자에게 물어본다: "반려 사유를 한 줄로 알려주세요."

2. 해당 Notion TODO:
   - Status를 "📋 대기"로 되돌린다
   - 작업 요약에 반려 사유를 추가한다: "🔄 수정 요청: {사유}"
   - 담당 = "Claude Code"

3. 또는 수정 범위가 다르면 새 TODO를 생성한다:
   - Task명: "수정: {원래 Task명} — {사유 요약}"
   - Priority: 원래 항목과 동일
   - Phase: 원래 항목과 동일
   - Status: "📋 대기"
   - 담당: "Claude Code"

4. 출력:
```
🔄 반려: {Task명}
사유: {사유}
→ 수정 TODO 생성됨. 다음 Loop에서 자동 처리됩니다.
```

5. 대기 TODO가 있으면: "📋 대기 {N}건. /kickoff로 바로 실행할 수 있습니다."
