# /weekly-report — 주간 리포트 생성

이번 주의 작업 현황을 정리하여 Notion 프로젝트 페이지에 주간 리포트를 추가합니다.

## 실행 순서

1. **Notion TODO DB 전체 조회**: 모든 TODO 항목의 Status, 시작일, 완료일을 확인합니다.

2. **이번 주 완료 항목 집계**: 완료일이 이번 주(월~금)인 항목을 추출합니다.

3. **진행 중 항목 확인**: Status가 "🔄 진행중" 또는 "👀 리뷰중"인 항목을 추출합니다.

4. **다음 주 예정 항목**: Status가 "📋 대기"이고 Priority가 높은 순으로 상위 5개를 추출합니다.

5. **이슈 항목 확인**: "⏸️ 보류" 상태이거나, "👀 리뷰중"인데 고객 확인이 안 된 항목을 추출합니다.

6. **리포트 작성**: 아래 형식으로 Notion 프로젝트 페이지의 커뮤니케이션 로그에 추가합니다.

## 리포트 형식

```markdown
## 📊 주간 리포트 — {YYYY년 MM월 DD일}

### ✅ 이번 주 완료 ({N}건)
- {Task명}: {작업 요약 1줄}
- {Task명}: {작업 요약 1줄}

### 🔄 현재 진행 중 ({N}건)
- {Task명} — {담당}: {현재 상태 간단 설명}

### 📋 다음 주 예정 ({N}건)
- {Task명} ({Priority})
- {Task명} ({Priority})

### ⚠️ 확인 필요 사항
- {고객 확인이 필요한 항목 또는 보류 사항}

### 📈 진행률
- 전체 TODO: {전체}건
- 완료: {완료}건 ({퍼센트}%)
- 진행중: {진행중}건
- 대기: {대기}건
```

## 주의사항

- 리포트는 **고객이 읽는 문서**이므로 기술 용어를 사용하지 않습니다
- 이슈 사항은 솔직하게 기록하되, 해결 방안도 함께 제시합니다
- 진행률 퍼센트는 고객에게 체감 가능한 지표입니다
